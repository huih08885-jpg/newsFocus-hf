## 1. 设计系统概述

### 1.1 设计原则

- **现代化**: 采用现代UI设计趋势，简洁美观
- **响应式**: 支持桌面端、平板和移动端
- **一致性**: 统一的视觉语言和交互模式
- **易用性**: 直观的操作流程，降低学习成本
- **性能优先**: 优化加载速度，提升用户体验

## 2. 布局设计

### 2.1 整体布局结构

```
┌─────────────────────────────────────────────────────────┐
│                    Header (固定顶部)                      │
│  Logo | 导航菜单 | 搜索框 | 用户信息                      │
├──────────────┬──────────────────────────────────────────┤
│               │                                          │
│   Sidebar     │          Main Content                    │
│   (固定左侧)   │          (可滚动区域)                     │
│               │                                          │
│   - 仪表板     │                                          │
│   - 新闻列表   │                                          │
│   - 数据分析   │                                          │
│   - 历史查询   │                                          │
│   - 设置      │                                          │
│               │                                          │
└──────────────┴──────────────────────────────────────────┘
```

### 2.2 Header组件设计

**尺寸**: 高度 64px，固定顶部

**内容**:
- **左侧**: Logo + 应用名称
- **中间**: 导航菜单（可选，移动端隐藏）
- **右侧**: 
  - 搜索框（可选）
  - 通知图标（带徽章）
  - 用户头像/菜单

**样式**:
- 背景: `--bg-primary`
- 边框: 底部 1px `--gray-200`
- 阴影: `--shadow-sm`
- 内边距: 水平 24px，垂直 16px

**响应式**:
- 桌面端: 显示完整内容
- 移动端: Logo + 菜单按钮（汉堡菜单）

### 2.3 Sidebar组件设计

**尺寸**: 宽度 256px（桌面端），固定左侧

**内容**:
- 导航菜单项（图标 + 文字）
- 折叠/展开按钮（可选）

**菜单项结构**:
```
📊 仪表板
📰 新闻列表
📈 数据分析
📅 历史查询
⚙️ 设置
```

**样式**:
- 背景: `--bg-primary`
- 边框: 右侧 1px `--gray-200`
- 高度: 100vh - 64px（Header高度）
- 滚动: 内容超出时可滚动

**响应式**:
- 桌面端: 固定显示
- 移动端: 抽屉式（Drawer），点击菜单按钮展开

### 2.4 Main Content区域

**样式**:
- 背景: `--bg-secondary`
- 内边距: 24px（桌面端），16px（移动端）
- 最大宽度: 1400px（居中）
- 滚动: 内容超出时可滚动

## 4. 组件详细设计

### 4.1 按钮组件

#### 4.1.1 主要按钮 (Primary Button)

**样式**:
- 背景: `--primary`
- 文字: 白色
- 圆角: `--radius-md`
- 内边距: 12px 24px
- 字重: `--font-medium`
- 悬停: 背景变深，阴影增强

**变体**:
- 次要按钮: 边框样式，透明背景
- 危险按钮: 红色背景
- 文本按钮: 无背景，仅文字

#### 4.1.2 图标按钮

**尺寸**: 40x40px（桌面端），36x36px（移动端）

**样式**:
- 圆形或圆角方形
- 悬停背景色变化
- 图标居中

### 4.2 卡片组件

**样式**:
- 背景: `--bg-primary`
- 圆角: `--radius-lg`
- 阴影: `--shadow-md`
- 内边距: 24px
- 边框: 1px `--gray-200`（可选）

**变体**:
- 可点击卡片: 悬停阴影增强，光标指针
- 静态卡片: 无交互效果

### 4.3 表格组件

**样式**:
- 背景: `--bg-primary`
- 圆角: `--radius-lg`
- 表头: 背景 `--gray-50`，字重加粗
- 行: 悬停背景 `--gray-50`
- 边框: 1px `--gray-200`

**响应式**:
- 桌面端: 完整表格
- 移动端: 卡片式布局

### 4.4 输入框组件

**样式**:
- 高度: 40px
- 边框: 1px `--gray-300`
- 圆角: `--radius-md`
- 内边距: 12px
- 焦点: 边框变蓝，阴影

**状态**:
- 默认: 灰色边框
- 焦点: 蓝色边框，阴影
- 错误: 红色边框
- 禁用: 灰色背景，不可编辑

### 4.5 徽章组件

**样式**:
- 内边距: 4px 8px
- 圆角: `--radius-full`
- 字体: 12px
- 字重: `--font-medium`

**变体**:
- 默认: 灰色背景
- 成功: 绿色背景
- 警告: 橙色背景
- 错误: 红色背景
- 信息: 蓝色背景

## 7. 动画和过渡

### 7.1 过渡效果

**页面切换**: 淡入淡出，200ms

**卡片悬停**: 阴影增强，150ms

**按钮点击**: 轻微缩放，100ms

**模态框**: 淡入 + 缩放，200ms

### 7.2 动画原则

- 快速: 不超过300ms
- 自然: 使用ease-out缓动
- 适度: 不过度使用动画

## 9. Google Stitch使用指南

### 9.1 导入设计规范

1. 在Google Stitch中创建新的设计系统
2. 导入颜色、字体、间距等设计Token
3. 创建组件库，复用基础组件

### 9.2 页面实现步骤

1. **创建页面框架**
   - Header组件
   - Sidebar组件
   - Main Content区域

2. **实现页面内容**
   - 根据页面布局添加组件
   - 使用设计系统中的组件
   - 保持间距和样式一致

3. **添加交互**
   - 设置悬停效果
   - 添加点击交互
   - 实现状态变化

4. **响应式适配**
   - 创建移动端视图
   - 调整布局和间距
   - 测试不同屏幕尺寸

### 9.3 组件复用

- 创建可复用组件库

- 保持组件一致性
